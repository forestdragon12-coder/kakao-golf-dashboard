# 산점도 재설계 기획

## 1. 동심원 겹침 처리

### 크기 규칙
- 모든 버블 크기 = 가격대에 비례 (고정, 겹쳐도 커지지 않음)
- 겹칠 때: 같은 크기 안에서 링 분할

### 겹침 표현
```
1개 코스: ⬤ (채움)
2개 겹침: ◎⬤ (중심 + 링)
3개 겹침: ◎◎⬤ (중심 + 링 + 바깥 링)
```

### 코스 순서 (기본)
중심→바깥: 광주CC → 르오네뜨 → 어등산 → 푸른솔장성 → 해피니스 → 골드레이크 → 무등산 → 베르힐(바깥)

### 선택 순서 반영
- 코스 해제→재선택 시 선택 순서대로 안→바깥 쌓임
- 마지막 선택 = 가장 바깥 링

## 2. 버블 상태 표현

| 상태 | 표현 |
|------|------|
| 코스 구분 | 색상 (기존) |
| 가격대 | 크기 (비쌀수록 큼) |
| 정가 | 채움 |
| 특가 | 반투명 |
| 인상 | ▲ 빨간 삼각형 (버블 위) |
| 인하 | ▼ 하늘색 삼각형 (버블 아래) |

## 3. 툴팁 (마우스 오버)

같은 D-day × 가격대의 모든 코스 정보:
```
┌─ D-2 / 10만원대 ──────────────┐
│  광주CC     8만→7만 ▼인하 3개  │
│  어등산     11만    정가  5개  │
│  베르힐     12만→13만 ▲인상 2개│
│  해피니스   5만     특가  8개  │
│  합계: 18슬롯 / 특가 8건      │
└───────────────────────────────┘
```

## 4. Y축 토글

### 가격 모드 (기본)
- Y축 = 가격 (만원)
- 기존 산점도

### 시간대 모드
- Y축 = 06시~16시 (아래=아침/저가, 위=오후/고가)
- 버블 크기 = 가격
- 색상 = 코스
