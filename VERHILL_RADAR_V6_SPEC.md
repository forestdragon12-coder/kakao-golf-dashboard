## 1. 용어 체계

| 용어 | 정의 |
|------|------|
| **신규오픈** | play_date가 수집 범위(D+30)에 처음 진입한 슬롯 |
| **티취소** | 어제 없었는데 오늘 나타난 슬롯 (예약취소 재오픈) |
| **소진** | 어제 있었는데 오늘 없는 슬롯 (예약됨) |
| **미판매** | 팔리지 않고 경기일이 지난 슬롯 (손실 매출) |
| **유지** | 어제도 오늘도 있는 슬롯 |
| **정가** | 슬롯의 최초 포착 가격 (골프장이 처음 설정한 가격) |
| **할인 강도** | (정가 - 현재가) / 정가 × 100 (양수=할인, 음수=프리미엄) |

**검증 공식:**
```
증감 = (신규오픈 + 티취소) - (소진 + 미판매)
```

## 2. 시간축 분리

### 원칙
```
과거 데이터 = 수집일(collected_date) → 기준일 ◀▶ 네비게이터
미래 데이터 = 경기일(play_date)     → 경기일 필터 [임박/1주/2주/전체]
두 개를 절대 섞지 않는다
```

### 탭 구성

|---|------|----------|----------|-----------|
| 1 | 브리핑 | 과거 | ✅ | ❌ |
| 2 | **판매내역** | 과거 | ✅ | ❌ |
| 3 | 소진 패턴 | 과거 | ✅ | ❌ |
| 4 | 가격 흐름 | 미래 | ✅ | ✅ 임박/1주/2주/전체 |
| 5 | 할인 반응 | 미래 | ✅ | ✅ |
| 6 | 수익 구조 | 과거 | ✅ | ❌ |
| 7 | 코스 비교 | 과거 | ✅ | ❌ |
| 8 | AI 진단 | 과거 | ✅ | ❌ |
| 9 | 티타임 상세 | 미래 | ❌ 항상 오늘 | ✅ |

### 탭별 설명 (산출 공식 포함)

| 탭 | 설명 |
|---|------|
| 브리핑 | 8개 골프장의 오늘 가격과 남은 티타임, 특가 정보. 소진 = 어제 있다 오늘 없는 슬롯 · 미판매 = 팔리지 않고 경기일 지난 슬롯 · 증감 = (신규+티취소) - (소진+미판매) |
| 판매내역 | 어제 대비 판매 현황 추적. 소진율 = 소진 슬롯 ÷ 전일 슬롯 × 100 |
| 가격 흐름 | 경기일(D-day)이 다가올수록 가격 변화. 정가 = 슬롯 최초 포착 가격 · 할인 강도 = (정가 - 현재가) ÷ 정가 × 100 |
| 수익 구조 | 정가(최초 포착가) 대비 실제 판매가 비율. 양수 = 할인, 음수 = 프리미엄 · 전량 특가 코스는 '-' 표시 |

## 6. 미판매 분석

매일 수집 시, 전날 경기의 미판매 슬롯을 unsold_slots 테이블에 기록.

| 지표 | 계산 |
|------|------|
| 미판매 수 | D-1 마지막 수집의 잔여 슬롯 |
| 미판매율 | 미판매 / 최초 오픈 슬롯 × 100 |
| 추정 손실액 | 미판매 슬롯 가격 합계 |
| 코스별 미판매 | 코스별 미판매 수 + 비율 |
| 특가 미판매 | 할인해도 안 팔린 슬롯 (수요 부재 구간) |

Tab1 브리핑에 미판매 현황 섹션으로 표시.

## 7. 특가 분류 체계

### 정가 기준
정가 = 슬롯의 **최초 포착 가격** (D+30에서 처음 수집된 가격).
폴백: 세그먼트 비할인 중앙값 → 코스 전체 비할인 중앙값.

### 특가 등급

| 등급 | 조건 | 표시 |
|------|------|------|
| **마케팅 특가** | 4주 평균 특가율 80% 이상 | 무시 (그냥 "가격"으로) |
| **할인 확대** | 전주 대비 특가율 +20%p 이상 | 강조 (전략 변화 시그널) |
| **할인 축소** | 전주 대비 특가율 -20%p 이상 | 시그널 (가격 자신감) |
| **실질 할인** | 정가 대비 10% 이상 차이 | 할인 태그 |
| **소폭 할인** | 정가 대비 10% 미만 차이 | 약하게 또는 무시 |

데이터 4주 미만 시 특가 비율만 표시, 등급 판정 보류.

## 10. 보안 체계

### 인증 + 등급

| 등급 | 세션 | 워터마크 | 캡처 | 만료 후 |
|------|------|---------|------|--------|
| 관리자 | 5시간 | 없음 | 허용 | 재로그인 |
| 매니저 | 5시간 | 옅은 8% | 허용 | 재로그인 |
| 뷰어 | 5시간 | 진한 20% | 차단 | 재로그인 |
| 게스트 | 첫 로그인 후 1시간 | 진한 20% | 차단+세션종료 | **영구 차단** |

- 실시간 세션 만료 (1분 간격 체크, 접속 중이라도 시간 지나면 로그아웃)
- 게스트: 첫 로그인 시각 기록 → 1시간 후 Firestore에서 active=false → 재접속 불가
- 관리자: 차단/삭제 버튼 비활성화 (자기 차단 방지)

### 관리자 패널 (프로필 이름 클릭으로 진입)
- **승인 대기**: 신청자 목록 → 뷰어/매니저/게스트로 승인 또는 거부
- **사용자 관리**: 등급, 표시 이름(워터마크용), 만료일, 접속 횟수, 세션 시간, 시간대 설정
- **접속 로그**: 이름 + 이메일 + 등급 + 행동 + 시간

### 사용자별 제한 설정

| 필드 | 예시 | 효과 |
|------|------|------|
| expires_at | 2026-03-31 | 기간 만료 후 차단 |
| starts_at | 2026-03-20 | 시작일 이전 접속 불가 |
| max_logins | 10 | 접속 횟수 제한 |
| session_hours | 3 | 세션 시간 개별 설정 |
| allowed_hours | [9, 18] | 09시~18시만 접속 |

### 기타 보안
- Firebase Auth (구글 로그인) + Firestore Security Rules + Storage Rules
- X-Frame-Options, CSP 헤더
- DevTools 감지 (viewer: 경고, guest: 세션종료)
- 인쇄/우클릭/드래그 차단 (viewer, guest)
- 워터마크: 관리자 지정 이름 + 접속시간 + 세션ID

