# 카카오골프 운영 자동화 정책
> 스케줄 / 저장 / 검증 기준 | 2026-03-13

## 1. 목적

이 문서는 수집, 보고서 생성, 저장/보관, 누적 데이터 검증의 운영 기준을 정의한다.

---

## 2. 일일 운영 스케줄

### 기본 원칙

- 수집은 매일 1회 이상 동일한 시간대에 수행한다.
- 보고서는 수집 직후 자동 생성한다.
- 일간 보고서는 매일 생성한다.
- 주간 보고서는 매주 1회 생성한다.
- 월간 보고서는 매월 1회 생성한다.
- 연간 보고서는 매년 1회 생성한다.

### 권장 스케줄

#### 매일
- 06:50 기기 기상
- 07:00 수집 실행
- 07:20 분석 파이프라인 완료 목표
- 07:25 일간 보고서 저장

#### 매주
- 매주 월요일 07:55 주간 보고서 생성
- 기준일은 실행일 당일

#### 매월
- 매월 1일 08:05 월간 보고서 생성
- 기준일은 해당 월 1일 실행 시점

#### 매년
- 매년 1월 2일 08:15 연간 보고서 생성
- 기준일은 실행일 당일

---

## 3. 보고서 생성 정책

### 일간 보고서

- 수집이 성공했으면 자동 생성
- 수집 데이터가 0건이어도 실패 리포트 생성 가능
- 텔레그램은 설정 시에만 발송

### 주간/월간/연간 보고서

- 수집과 분리된 보고서 모드로 생성
- 누적 데이터가 부족해도 반드시 파일은 생성
- 해석이 부족한 경우 risks에 명시

---

## 4. 저장 정책

### 저장 경로

- 보고서: `reports/`
- 로그: `logs/`
- DB: `data/golf.db`

### 파일 규칙

- 일간: `daily_report_YYYY-MM-DD.txt`
- 주간: `weekly_report_YYYY-MM-DD.txt`
- 월간: `monthly_report_YYYY-MM-DD.txt`
- 연간: `yearly_report_YYYY-MM-DD.txt`

### 향후 확장

- `reports/daily/`
- `reports/weekly/`
- `reports/monthly/`
- `reports/yearly/`

형태로 분리 가능

## 5. 보관 정책

### 권장 보관 기간

- 일간 보고서: 90일
- 주간 보고서: 1년
- 월간 보고서: 3년
- 연간 보고서: 영구 보관 권장
- 로그 파일: 30일

### 정리 원칙

- 텍스트 보고서는 삭제 전 압축/백업 고려
- 연간 보고서는 절대 자동 삭제하지 않음
- DB는 정기 백업 대상

## 6. 누적 데이터 검증 체크리스트

### 6.1 수집 품질

- 8개 골프장 모두 당일 수집되었는가
- 특정 골프장 수집량이 비정상적으로 적지 않은가
- 디버그 이미지/텍스트가 과도하게 쌓이지 않는가

### 6.2 가격 변동 품질

- 전일 대비 `price_change_events`가 생성되는가
- 하루치만 있을 때 0건으로 나오는가
- 동일 슬롯 비교가 정상 동작하는가

### 6.3 할인 반응 품질

- `price_response_metrics`가 D+1/D+3/D+5 기준으로 채워지는가
- 수집 누락일을 반응으로 오판하지 않는가
- 강함/보통/약함/없음 분류가 정상인가

### 6.4 반복 패턴 품질

- 반복 약세는 `distinct report_date` 기준으로 계산되는가
- 하루치 데이터에서 구조적 약세가 과대판정되지 않는가
- 최소 관측일수 기준이 지켜지는가

### 6.5 보고서 품질

- action과 reason이 서로 충돌하지 않는가
- risks가 데이터 부족을 숨기지 않는가
- weekly/monthly/yearly가 부분 데이터 상태를 명시하는가

### 6.6 LLM 품질

- LLM 미설정 시 deterministic fallback이 동작하는가
- LLM 응답이 숫자를 바꾸지 않는가
- payload 밖 사실을 추가하지 않는가
