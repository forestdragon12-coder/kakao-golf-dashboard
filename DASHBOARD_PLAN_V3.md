## 0. 설계 전제

### 데이터 특성
- 매일 07:00 수집, D+1~D+30 경기일 대상, 8개 골프장
- 수집일 기준 2일차 (2026-03-13~14). 향후 매일 누적
- 매진 슬롯은 카카오 페이지에서 제거됨 → 잔여 COUNT = available_slots
- 슬롯 소진 = 전일 대비 slot_identity_key 집합 차이로 계산

### 핵심 보정 사항 5건

| # | 문제 | 해법 |
|---|------|------|
| 1 | 스크래퍼 장애 시 전량 소진으로 오판 | 전일 대비 70%+ 급감 = scraper_failure_flag, 소진 계산 제외 |
| 2 | Yield 기준가로 최초 관측가 사용 불가 | 세그먼트 기대가격(비할인 슬롯 중앙값) 사용 |
| 3 | 비할인 비교군 부재 (어등산 80/81 할인) | D-day 곡선 잔차법으로 할인 효과 추정 |
| 4 | course_sub 불안정 (날짜별 표기 변동) | course_variant 정규화 사전 적용 |
| 5 | 순소진 vs 총소진 혼동 | slot_identity_key 매칭: gross_consumed + gross_new 분리 |

### 가격 인상 특수 처리 원칙
- 골프장 업계에서 비현실적 고가 초기 설정 전략은 사용되지 않음 → 무시
- 가격 인상은 특수 상황이므로 일반 인하보다 상세하게 표기
- 표기 항목: 인상 감지일시, 해당 경기일, 시간대(1부/2부), 서브코스, 인상폭(원/%), D-day
- 목적: 운영진이 "왜 올렸는지" 검토 가능하도록 맥락 제공

## 2. 공통 UI 컴포넌트

### 2.1 날짜 구간 선택기 (듀얼 축)

두 개의 독립 날짜 축이 필요하다.

| 축 | 의미 | 용도 |
|----|------|------|
| 수집일(collected_date) | 데이터를 관측한 날 | "어제 대비 변화", "수집 건강도" |
| 경기일(play_date) | 실제 라운드 날 | "4/5 토요일 상황", "D-day별 추이" |

설계:
- 상단 글로벌 필터 영역에 두 축 각각의 날짜 범위 선택기 배치
- 기본값: 수집일 = 최근 7일, 경기일 = 향후 14일
- 탭별로 어느 축이 주축인지 명시 (Tab 1은 수집일 주축, Tab 2는 경기일 주축)
- 단일 날짜 선택 / 범위 선택 모두 지원
- 데이터 축적 30일 이후: 월간/주간 프리셋 버튼 추가

### 2.2 글로벌 필터

- 골프장 선택 (멀티)
- 서브코스 선택 (선택된 골프장 기준 동적)
- 요일 유형 (평일/토/일)
- 파트 (1부/2부)
- 회원구분 (대중제/회원제/전체) — 골드레이크·해피니스만 활성
- 할인 상태 (할인/비할인/전체)

### 2.3 데이터 축적 대응 전략

| 축적 기간 | 표현 방식 |
|-----------|-----------|
| ~7일 | 일별 포인트 그래프, 테이블 중심 |
| 7~30일 | 일별 라인차트 + 주간 요약 KPI |
| 30~90일 | 주간 집계 라인 + 월간 요약 |
| 90일+ | 월간 집계, 시즌 비교, YoY |

원칙: 축적 기간에 따라 grain을 자동 전환하되, 드릴다운으로 원시 단위 접근 가능

## 3. Tab 1 — 오늘의 브리핑

### 목적
오늘(최신 수집일) 기준으로 전일 대비 핵심 변화를 한 눈에 파악

### 구성

**KPI 카드 (상단)**
- 총 관측 슬롯 수 (전일 대비 ±)
- 신규 소진 슬롯 수 (gross_consumed)
- 가격 변경 이벤트 수 (인하/인상 분리)
- 신규 프로모 부착 수
- 스크래퍼 건강도 (정상/경고/장애)

**경보 섹션**
- 스크래퍼 장애 의심 (70%+ 급감 골프장)
- 대량 인하 감지 (단일 골프장 5건+ 인하)
- 회원제 오픈 감지 (골드레이크/해피니스)
- 공급 충격 (전일 대비 공급량 ±30% 이상)

**가격 변경 이벤트 테이블**

일반 인하 표기:
| 골프장 | 서브코스 | 경기일 | 시간 | 변경전 | 변경후 | 변동폭 | D-day |
|--------|---------|--------|------|--------|--------|--------|-------|

가격 인상 특수 표기 (별도 섹션, 강조색):
| 골프장 | 서브코스 | 경기일 | 시간 | 시간대 | 요일 | 변경전 | 변경후 | 인상폭 | 인상률 | D-day | 감지일시 | 맥락 메모 |
|--------|---------|--------|------|--------|------|--------|--------|--------|--------|-------|---------|-----------|

인상 맥락 메모 자동 생성 규칙:
- D-day ≤ 3 + 잔여 적음 → "직전 수요 강세 반영 추정"
- 주말 경기일 + 1부 → "주말 프라임타임 가격 조정"
- 동일 골프장 다른 서브코스도 동시 인상 → "전체 가격 재조정"
- 회원제 오픈 직후 → "회원제 오픈 연동 가격 조정"
- 해당 없음 → "원인 미상 — 운영진 확인 필요"

**소진 현황 요약 (골프장별)**
- 골프장 | 전일 잔여 | 금일 잔여 | 순소진 | 신규등장 | 총소진 | 소진율

## 7. Tab 5A — 할인 반응 분석

### 목적
할인이 실제 소진을 앞당겼는지, D-day 자연 가속과 분리하여 측정.
나아가 할인이 만든 **증분 소진량**을 정량화하고, 그 증분의 **매출 기여**까지 산출.

### D-day 교란 보정 방법

문제: 할인 시점이 경기일에 가까울수록 자연 예약 가속(D-day proximity effect)과 할인 효과가 뒤섞임

해법: D-day 정규화 기준선 대비 Lift 계산

```
baseline_consumption_rate(d) = 해당 세그먼트의 D-day=d일 때 비할인 평균 소진속도
actual_consumption_rate = 할인 적용 후 실제 소진속도
lift = actual / baseline(same_d) - 1
```

비할인 비교군 부재 시 (어등산 등):
```
C안: D-day 곡선 잔차법
1. 전체 데이터로 D-day 소진 곡선 피팅
2. 할인 이벤트 시점의 잔차(실제 - 예측) = 할인 순효과 추정
3. 60일+ 데이터 축적 후 시계열 기준선으로 전환
```

### 증분 소진 정량화 (Incremental Volume)

할인 후 소진된 슬롯은 세 가지 범주로 분리된다:

| 범주 | 정의 | 매출 영향 |
|------|------|-----------|
| A. 잠식 슬롯 (Cannibalized) | 할인 없어도 팔렸을 슬롯. 할인가에 팔림 | 순손실 = (기대가 - 할인가) × 수량 |
| B. 증분 슬롯 (Incremental) | 할인 덕에 추가 팔린 슬롯 | 순이득 = 할인가 × 수량 |
| C. 미소진 슬롯 (Unsold) | 할인해도 안 팔린 슬롯 | 매출 0 |

계산:
```
measurement_window = 할인 적용 후 1~3일

actual_consumed     = 측정 구간 내 실제 소진 슬롯 수
baseline_consumed   = D-day 기준선 예측 "할인 없었으면 소진됐을" 슬롯 수

incremental_slots   = max(0, actual_consumed - baseline_consumed)
cannibalized_slots  = min(baseline_consumed, actual_consumed)
```

### 매출 환산 (Revenue Attribution)

```
segment_expected_price = 세그먼트 기대가격 (비할인 중앙값)
avg_discount_price     = 할인 이벤트의 평균 소진 가격

# 증분 매출: 할인이 만든 추가 매출
incremental_revenue = incremental_slots × avg_discount_price

# 잠식 비용: 정가에 팔 수 있었는데 할인가에 판 차액
cannibalization_cost = cannibalized_slots × (segment_expected_price - avg_discount_price)

# 순 매출 영향: 할인의 진짜 손익
net_revenue_impact = incremental_revenue - cannibalization_cost

# 할인 ROI: 잠식 비용 1원당 순매출 영향
discount_roi = net_revenue_impact / cannibalization_cost
```

판정 기준:
- net_revenue_impact > 0, ROI > 1 → 할인이 이득. 증분 매출이 잠식을 초과
- net_revenue_impact > 0, ROI < 1 → 할인이 이득이지만 효율 낮음
- net_revenue_impact ≤ 0 → 할인이 손해. 안 깎는 게 나았다
- incremental_slots = 0 → 할인 효과 없음. 순수 잠식만 발생

### 예시

```
광주CC 동악 토요일 1부 | 세그먼트 기대가 130,000원 | 할인가 110,000원 (D-7)

Case A — 할인이 이득:
  실제 소진 8 / 기준선 5 → 증분 3, 잠식 5
  증분매출 = 3 × 110,000 = 330,000
  잠식비용 = 5 × 20,000  = 100,000
  순영향 = +230,000원 (ROI 2.3) ← 할인 효과 우수

Case B — 겨우 본전:
  실제 소진 6 / 기준선 5 → 증분 1, 잠식 5
  증분매출 = 110,000
  잠식비용 = 100,000
  순영향 = +10,000원 (ROI 0.1) ← 할인 비효율

Case C — 순손실:
  실제 소진 5 / 기준선 5 → 증분 0, 잠식 5
  증분매출 = 0
  잠식비용 = 100,000
  순영향 = -100,000원 ← 안 깎는 게 나았다
```

### 시각화

**메인: 할인 이벤트별 Lift 차트**
- X축: 할인 이벤트 (시간순)
- Y축: Lift (%)
- 색상: 효과 등급 (강함 >50% / 보통 20-50% / 약함 0-20% / 역효과 <0%)
- 라벨: 골프장, D-day, 할인폭

**메인: 할인 이벤트별 워터폴 차트 (매출 영향)**
- 각 할인 이벤트당 하나의 워터폴:
  [기대매출] →(-잠식비용)→ [잠식 후] →(+증분매출)→ [실제매출]
- 순영향 양수 = 녹색, 음수 = 적색
- 이벤트 간 비교로 "어떤 할인이 실제로 돈이 됐는지" 한 눈에 파악

**보조: 증분 슬롯 수 vs 할인 깊이 산개도**
- X축: 할인 깊이 (%)
- Y축: 증분 슬롯 수
- 점 크기: 순 매출 영향 (절대값)
- 점 색상: 양수(녹)/음수(적)
- 인사이트: "몇 % 깎아야 추가 소진이 생기는가"의 임계점

**보조: D-day별 할인 효과 분포**
- X축: D-day
- Y축: Lift
- 인사이트: D-day 가까울수록 할인 효과가 작아지는지 확인

**보조: 골프장별 할인 효과 요약**
- 바 차트: 골프장별 평균 Lift
- 할인 빈도 vs 효과 사분면 (많이 하고 효과 좋음 / 많이 하고 효과 없음 등)

**보조: 골프장별 할인 ROI 요약 테이블**
| 골프장 | 할인 이벤트 수 | 총 증분 슬롯 | 총 잠식 슬롯 | 증분매출 합계 | 잠식비용 합계 | 순매출영향 | 평균 ROI |

### 신뢰도 표기

| 등급 | 조건 | 표기 |
|------|------|------|
| High | 비할인 비교군 30건+, 기준선 R²>0.7 | 수치 그대로 표기 |
| Medium | 비교군 10-30건 또는 R²<0.7 | 수치 + "추정" 라벨 |
| Low | 비교군 <10건, 잔차법 사용 | "참고용" 회색 표기 |

최소 관측 수 미달 시 "데이터 부족" 표기, 계산 자체를 보류

## 8. Tab 5B — 수익 구조 분석

### 목적
슬롯 단위 수익 구조를 분석. Yield, 할인 비용, 소진 달성률을 통해 "돈이 어디서 새는가"를 파악

### 지표 정의

**Yield (소진 달성률)**
```
yield = 소진시점가격 / 세그먼트 기대가격
세그먼트 기대가격 = 동일 (course_id, part_type, weekday_type, season)의
                    비할인 슬롯 가격 중앙값
```
- yield > 1.0 → 프리미엄 소진
- yield = 1.0 → 정상가 소진
- yield < 1.0 → 할인 소진 (수치가 낮을수록 할인 깊음)

주의: 최초 관측가 ≠ 정상가. 골프장 업계에서 비현실적 고가 초기 설정은 사용 안 함 (확인 완료)

**할인 비용 (Discount Cost)**
```
discount_cost_per_slot = 세그먼트 기대가격 - 실제 소진가격
total_discount_cost = Σ(discount_cost_per_slot) for 할인 소진 슬롯
```

**할인 소진 비율**
```
discount_sell_ratio = 할인 상태 소진 슬롯 수 / 전체 소진 슬롯 수
```

**할인 효율 지표 (Tab 5A 연동)**
```
revenue_per_discount_event = 순매출영향 / 할인 이벤트 수
cost_per_incremental_slot = 잠식비용 / 증분 슬롯 수
```

### 시각화

**메인: Yield 분포 (히스토그램)**
- X축: Yield (0.5 ~ 1.5)
- Y축: 슬롯 수
- 색상: 골프장별
- 기준선: yield = 1.0

**보조: 골프장별 수익 구조 요약 테이블**
| 골프장 | 총 소진 | 할인 소진 | 비할인 소진 | 증분 소진 | 평균 Yield | 총 할인비용 | 순매출영향 | 할인소진비율 |

**보조: 시간대별 Yield 히트맵**
- X축: 요일
- Y축: 시간대
- 셀: 평균 Yield
- 인사이트: 어느 구간에서 수익이 새는지

**보조: 할인 깊이 vs 소진 속도 산개도**
- X축: 할인 깊이 (%)
- Y축: 소진 소요 일수
- 인사이트: "얼마나 깎아야 팔리는가"의 효율 곡선

**보조: 수익 누수 트렌드 (시계열)**
- X축: 수집일 (누적)
- Y축: 누적 할인 비용 / 누적 증분 매출
- 두 라인의 갭 = 순손익 추이
- 인사이트: 시간이 갈수록 할인 의존도가 늘고 있는지, 줄고 있는지

용도: Tab 3 소진 패턴 매트릭스의 시간대 블록 기준으로 활용

### 12.3 스크래퍼 건강도 체크

수집 완료 후 자동 실행:
```python
def check_scraper_health(today_counts, yesterday_counts):
    for course in courses:
        ratio = today_counts[course] / yesterday_counts[course]
        if ratio < 0.3:
            flag_scraper_failure(course, today)
```
