## 0. 설계 전제

### 데이터 특성
- 매일 07:00 수집, D+1~D+30 경기일 대상, 8개 골프장
- 수집일 기준 2일차 (2026-03-13~14). 향후 매일 누적

- **카카오골프 = 해당 골프장의 전체 예약 채널로 간주** (운영 판단의 주 근거로 사용)

### 가격 인상 특수 처리 원칙
- 골프장 업계에서 비현실적 고가 초기 설정 전략은 사용되지 않음 → 무시
- 가격 인상은 특수 상황이므로 일반 인하보다 상세하게 표기
- 표기 항목: 인상 감지일시, 해당 경기일, 시간대(1부/2부), 서브코스, 인상폭(원/%), D-day
- **추가: 인상 시점 전후 7일간 경쟁사 가격 변화를 자동 병기** (시장 전체 움직임 vs 독자 판단 구분)
- 목적: 운영진이 "왜 올렸는지" 검토 가능하도록 맥락 제공

### 세그먼트 기대가격 폴백 체인

Yield, 할인 비용, 증분 분석의 기준이 되는 segment_expected_price가
계산 불능 상태에 빠지지 않도록 계단식 폴백 적용:

```
1순위: 현재 세그먼트 비할인 슬롯 중앙값 (표본 5건+)
2순위: 직전 7일 동일 세그먼트 비할인 중앙값
3순위: 직전 30일 동일 세그먼트 비할인 중앙값
4순위: 동일 시즌 + 동일 요일유형 전체 기간 비할인 중앙값
5순위: 계산 불가 → "기준가 미확정" 표기, 관련 지표 회색 처리
```
각 단계에서 사용된 폴백 레벨을 UI에 표기 (예: "기준가: 130,000원 [7일 평균]")

### 회원제 슬롯 미오픈 처리 원칙 (전 분석 공통)

**적용 대상**: 골드레이크, 해피니스 (회원제/대중제 혼합 구조)
**미오픈 정의**: 회원제 서브코스가 카카오에 한 번도 등장하지 않은 수집일

회원제 슬롯이 카카오에 안 보이는 이유는 두 가지다:
- 아직 회원 전용 예약 기간 (일반 공개 전)
- 이미 회원 예약으로 전량 소진

두 경우 모두 카카오 데이터만으로 구분 불가. 따라서:

```
슬롯 상태 4종:
  ACTIVE      = 현재 카카오에 노출 중
  CONSUMED    = 이전 관측 후 사라짐 (예약 완료로 간주)
  REAPPEARED  = 소진 후 재등장 (취소 복귀)
  NOT_OPEN    = 회원제 코스의 해당 경기일 슬롯이 아직 미등장
               (오픈 전 상태 — 소진과 구분)
```

NOT_OPEN 판정 조건:
```python
def is_not_open(course, course_variant, play_date, collected_date):
    # 해당 course_variant가 해당 play_date에 대해
    # 지금까지 단 한 번도 관측된 적 없고
    # 해당 골프장이 회원제 혼합 구조인 경우
    return (
        course in MEMBER_COURSES and
        course_variant in MEMBER_COURSES[course]["회원제"] and
        no_prior_observation(course, course_variant, play_date)
    )
```

**NOT_OPEN이 영향받는 모든 분석 영역과 처리 방식:**

| 분석 영역 | 문제 | 처리 방식 |
|----------|------|-----------|
| 소진율 | 미오픈 기간 포함 시 소진 불가 구간이 0%로 왜곡 | 분모에서 NOT_OPEN 기간 제외 |
| D-day 소진 기준선 | 미오픈 기간 데이터가 곡선에 섞여 기준선 하향 왜곡 | NOT_OPEN 관측일 기준선 피팅에서 제외 |
| 공급 충격 감지 | 회원제 오픈 시 슬롯 급증 = 공급 충격 오판 | member_open_event로 분류, 충격 경보 제외 |
| 취소 추적 | 미오픈 → 오픈 = 재등장(취소복귀)으로 오판 | NOT_OPEN 이후 첫 등장 = first_seen으로 처리 |
| 스크래퍼 건강도 | 회원제 슬롯 없음 = 장애 오판 | 대중제 슬롯만으로 건강도 판정 |
| 할인 Lift 계산 | 오픈과 할인 동시 발생 시 효과 혼동 | 오픈 후 3일 이내 Lift = 신뢰도 Low 강제 |
| Yield / 증분·잠식 | 미오픈 기간 세그먼트 기대가격 표본 부족 | 폴백 체인 적용, 회원제 전용 기대가격 별도 관리 |
| 경쟁 비교 | 회원제 포함 골프장과 단일구조 골프장 직접 비교 왜곡 | 비교 시 대중제 슬롯만 사용, 회원제는 별도 라인 |
| 소진 패턴 히트맵 | 미오픈 요일×시간대 = "비인기" 오분류 | NOT_OPEN 셀 회색 + "미오픈" 툴팁 |
| AI 진단 룰 | 미오픈 상태에서 룰 A~J 판정 시 오진 | 모든 룰 판정 전 NOT_OPEN 여부 선행 확인 |

**UI 표기 원칙:**
- NOT_OPEN 구간: 회색 셀 / 회색 바 / 대시(—) 표기
- 오픈 감지 시: "🔓 회원제 오픈" 이벤트 마커 표시
- 오픈 후 데이터 축적 전: 신뢰도 Low 표기

### 악천후 유형별 처리 원칙

악천후는 종류마다 **영향 방향이 다르다.** 단순 "악천후 플래그"로 묶으면 분석이 왜곡된다.

```python
def classify_bad_weather(forecast):
    if forecast.rain_prob >= 60:
        return 'rain'       # 취소 증가 → 공급 ↑, 수요 ↓
    if forecast.temp_high >= 35:
        return 'heat'       # 오후 수요 급감, 오전은 영향 적음
    if forecast.wind_speed >= 10:
        return 'wind'       # 전 시간대 기피
    if forecast.temp_high <= 2:
        return 'cold'       # 새벽·오전 기피, 동결 위험
    if forecast.weather_code in SEVERE:
        return 'severe'     # 전면 운영 불가 수준
    return None
```

| 유형 | 영향 시간대 | 예상 효과 | Tab 1 경보 |
|------|-----------|----------|-----------|
| 우천 ⛈ | 전 시간대 | 취소↑ 재등장↑ 소진↓ | "강수 예보 — 취소 증가 대비" |
| 폭염 🌡 | **오후 2부 집중** | 오후 수요 급감, 오전은 영향 적음 | "폭염 예보 — 오후 2부 소진 둔화 예상" |
| 강풍 💨 | 전 시간대 | 전반적 기피 | "강풍 예보 — 전 시간대 위축" |
| 한파 ❄ | **새벽·오전 1부** | 새벽 특히 기피 | "한파 예보 — 1부 소진 약화 예상" |
| 심각 🚨 | 전 시간대 | 운영 불가 수준 | "심각 기상 — 당일 전량 소진 분석 제외" |

**분석 보정 방식:**
- 우천/강풍/한파/심각: 해당 경기일 소진 분석 = 악천후 그룹 분리, 기준선 계산 제외
- 폭염: **시간대별 분리** — 오전 1부는 정상 계산, 오후 2부만 폭염 그룹 처리
- Tab 3 히트맵: 악천후 셀에 유형별 아이콘 + 괄호 표기 "72% (폭염)" / "45% (우천)"

### 3.4 평일/주말 구조적 분리

평일과 주말은 필터 옵션이 아니라 **분석의 기본 단위**다.
가격 구조, 고객층, 할인 전략, 경쟁 양상이 완전히 다르기 때문이다.

| 구분 | 평일 (월~금) | 주말 (토~일) |
|------|-------------|-------------|
| 핵심 과제 | 안 팔리는 걸 파는 싸움 | 팔리는 걸 더 비싸게 파는 싸움 |
| 할인 의미 | 소진 자체가 목표 | 불필요한 할인 = 순수 손실 |
| 프리미엄 여력 | 낮음 | 높음 (특히 토 오전 1부) |
| 비교 기준 | 평일끼리 비교 | 주말끼리 비교 |

적용 방식:
- **뷰 모드 토글**: [전체] [평일] [주말] — 상단에 항상 표시
- 기본값: [전체] (양쪽 나란히 비교 뷰)
- [평일] 또는 [주말] 선택 시: 해당 유형만 표시, 지표 재계산
- 영향받는 탭: Tab 1, 3, 4, 5A, 5B, 7 (Tab 2, 6, 8은 개별 데이터라 필터로 충분)

전체 뷰일 때 표기 방식:
- KPI 카드: 평일/주말 나란히 2열 표기

## 5. 취소(재등장) 슬롯 구분

### 5.1 문제

slot_identity_key가 어제 있고 → 오늘 없으면 "소진"으로 판정한다.
그런데 내일 같은 key가 다시 나타나면? 그건 취소 후 재등장이다.
현재 설계에서 이것이 gross_new에 섞여서 "신규 오픈"과 구분이 안 된다.

### 5.2 3상태 분류

각 slot_identity_key의 관측 이력을 추적하여 상태를 분류:

```
상태 A: 신규 등장 (first_seen)
  - 해당 key가 전체 이력에서 처음 관측
  - 의미: 골프장이 새로 열었거나, 첫 수집 시점의 기존 슬롯

상태 B: 재등장 (reappeared / 취소 복귀)
  - 해당 key가 과거에 관측된 적 있고 → 한 번 이상 미관측 → 다시 관측
  - 의미: 예약 후 취소되어 다시 열림

상태 C: 소진 (consumed)
  - 해당 key가 관측되다가 → 미관측으로 전환 (아직 재등장 안 함)
  - 의미: 예약 완료 (단, 향후 재등장하면 B로 소급 변경)
```

### 5.3 구현

```python
def classify_slot_transition(key, observation_history):
    """
    observation_history: [True, True, False, True, ...]
    (수집일별 관측 여부 시계열)
    """
    appearances = [i for i, v in enumerate(observation_history) if v]
    disappearances = [i for i, v in enumerate(observation_history) if not v]

    if len(appearances) == 1 and appearances[0] == len(observation_history) - 1:
        # 오늘 처음 등장
        if key in historical_keys:
            return 'reappeared'  # 과거에 본 적 있음 = 취소 복귀
        return 'first_seen'     # 완전 신규

    # 관측 → 미관측 → 관측 패턴이 있으면 취소 이력 있음
    gaps = find_gaps(observation_history)
    if gaps:
        return 'reappeared'
    ...
```

## 10. Tab 5A — 할인 반응 분석

### 목적
할인이 실제 소진을 앞당겼는지, D-day 자연 가속과 분리하여 측정.
증분 소진량을 정량화하고, 매출 기여까지 산출.

### D-day 교란 보정 방법

```
baseline_consumption_rate(d) = 해당 세그먼트의 D-day=d일 때 비할인 평균 소진속도
actual_consumption_rate = 할인 적용 후 실제 소진속도
lift = actual / baseline(same_d) - 1
```

비할인 비교군 부재 시:
```
D-day 곡선 잔차법:
1. 전체 데이터로 D-day 소진 곡선 피팅
2. 할인 이벤트 시점의 잔차(실제 - 예측) = 할인 순효과 추정
3. 60일+ 데이터 축적 후 시계열 기준선으로 전환
```

### measurement_window: D-day 비례 설계

고정 1~3일이 아니라 D-day에 비례:

```
measurement_window = max(1, min(5, d_day // 5))

D-day 25~30 → window 5일 (효과가 천천히 나타남)
D-day 15~24 → window 3일
D-day 5~14  → window 2일
D-day 1~4   → window 1일 (급박한 시점, 당일 반응이 전부)
```

추가: 소진 속도가 기준선으로 복귀하는 시점을 감지하면 window를 자동 단축
(효과가 2일 만에 소멸했으면 3일치 볼 필요 없음)

### 증분 소진 정량화

| 범주 | 정의 | 매출 영향 |
|------|------|-----------|
| A. 잠식 (Cannibalized) | 할인 없어도 팔렸을 슬롯 | 손실 = (기대가 - 할인가) × 수량 |
| B. 증분 (Incremental) | 할인 덕에 추가 팔린 슬롯 | 이득 = 할인가 × 수량 |
| C. 미소진 (Unsold) | 할인해도 안 팔린 슬롯 | 매출 0 |

```
actual_consumed     = 측정 구간 내 실제 소진
baseline_consumed   = D-day 기준선 예측 소진
incremental_slots   = max(0, actual - baseline)
cannibalized_slots  = min(baseline, actual)
```

**악천후 보정**: 악천후 경기일의 할인 이벤트는 별도 그룹
- UI에서 ⛈ 표기, 일반 이벤트와 분리 집계
- 악천후 시 소진 기준선 자체가 낮으므로 별도 기준선 필요
  (초기에는 데이터 부족으로 제외 처리, 60일+ 후 악천후 전용 기준선 구축)

### 매출 환산

```
incremental_revenue  = incremental_slots × avg_discount_price
cannibalization_cost = cannibalized_slots × (segment_expected_price - avg_discount_price)
net_revenue_impact   = incremental_revenue - cannibalization_cost
discount_roi         = net_revenue_impact / cannibalization_cost
```

판정:
- net > 0, ROI > 1 → 할인 이득 (증분이 잠식 초과)
- net > 0, ROI < 1 → 이득이지만 효율 낮음
- net ≤ 0 → 할인 손해
- incremental = 0 → 효과 없음, 순수 잠식

### 시각화

**메인: 할인 이벤트별 Lift 차트**
- X축: 이벤트 (시간순), Y축: Lift (%)
- 색상: 효과 등급 (강함/보통/약함/역효과)

### 지표 정의

**Yield (소진 달성률)**
```
yield = 소진시점가격 / 세그먼트 기대가격
```
- yield > 1.0 → 프리미엄 소진
- yield = 1.0 → 정상가 소진
- yield < 1.0 → 할인 소진

세그먼트 기대가격은 §0 폴백 체인 적용. 사용된 폴백 레벨 표기.

**할인 비용 / 할인 소진 비율 / 할인 효율** — v3과 동일

### 시각화

**메인: Yield 분포**
- 히스토그램, 골프장별 색상, yield=1.0 기준선
- **평일/주말 분할 표시** (좌: 평일, 우: 주말) 또는 토글

**보조: 수익 구조 요약 테이블**
| 골프장 | 유형 | 총소진 | 할인소진 | 비할인소진 | 증분 | 평균Yield | 할인비용 | 순매출영향 |
