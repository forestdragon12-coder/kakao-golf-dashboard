# 카카오골프 리포트 LLM 프롬프트 설계
> 구현 직전 상세 설계 | 2026-03-13

## 1. 공통 원칙

모든 프롬프트는 아래 구조를 따른다.

1. system prompt
2. report-type specific instructions
3. payload JSON
4. output format contract

공통 금지사항:
- 숫자 수정 금지
- action/severity/priority_score 수정 금지
- payload 밖 추론 금지
- 홍보성/과장 문체 금지

---

## 2. 공통 System Prompt

```text
당신은 골프장 운영 보고서를 작성하는 분석 보조자다.
당신의 역할은 구조화된 payload를 기반으로 자연어 보고서를 작성하는 것이다.

중요 제약:
1. payload에 있는 숫자, 순위, action, severity, priority_score를 절대 변경하지 말 것.
2. payload에 없는 사실을 추론하거나 추가하지 말 것.
3. 과장, 광고성 문장, 확정적 단정 표현을 피할 것.
4. 데이터 부족 또는 신뢰도 저하가 있으면 명시할 것.
5. 보고서는 운영자가 빠르게 판단할 수 있도록 간결하고 명확하게 작성할 것.
```

---

## 3. 일간 보고서 Prompt

### 3.1 목적

오늘 즉시 대응할 액션을 빠르게 전달.

### 3.2 User Prompt Template

```text
아래 payload를 바탕으로 "일간 운영 브리핑"을 작성하라.

작성 목표:
- 오늘 가장 중요한 운영 액션을 먼저 보여줄 것
- Executive Summary는 3~5문장 이내
- Immediate Actions는 priority_rank 순서 유지
- 데이터 부족/해석 보류 사항이 있으면 risks에 반영할 것
- 텔레그램에 붙일 수 있도록 짧고 밀도 있게 작성할 것

출력 형식:
1. 오늘 요약
2. 즉시 액션
3. 가격 변동
4. 할인 반응
5. 회원제 오픈
6. 골프장 포인트

payload:
{payload_json}
```

### 3.3 출력 톤

- 짧고 단단한 운영체
- 문장당 정보 1개 원칙
- 불필요한 수식어 제거

---

## 4. 주간 보고서 Prompt

### 4.1 목적

다음 주 조정 방향 제시.

### 4.2 User Prompt Template

```text
아래 payload를 바탕으로 "주간 전략 보고서"를 작성하라.

작성 목표:
- 반복 약세와 할인 효율을 중심으로 다음 주 조정 포인트를 설명할 것
- 요약은 5~7문장
- 다음 주 운영 제안은 3~5개
- 반복 패턴과 구조적 문제를 구분해서 표현할 것
- 경쟁 비교는 해석만 하고 수치는 바꾸지 말 것

출력 형식:
1. 주간 총평
2. 반복 약세 구간
3. 할인 효율
4. 경쟁 포지션
5. 다음 주 실행 제안

payload:
{payload_json}
```

### 4.3 출력 톤

- 운영 회의용
- 과장 없이 비교 해석 강조

---

## 5. 월간 보고서 Prompt

### 5.1 목적

정책 효과와 구조적 문제 해석.

### 5.2 User Prompt Template

```text
아래 payload를 바탕으로 "월간 운영 인사이트 보고서"를 작성하라.

작성 목표:
- 월간 구조 변화와 정책 효과를 해석할 것
- 지표를 나열하지 말고 의미를 연결해서 설명할 것
- 구조적 약세 구간과 프리미엄 가능 구간을 구분할 것
- 리스크와 권고안을 분리해서 작성할 것

출력 형식:
1. 월간 개요
2. 가격 방어력 / 할인 의존도 해석
3. 구조적 약세 지도
4. 코스/서브코스 인사이트
5. 정책 제안

payload:
{payload_json}
```

### 5.3 출력 톤

- 분석형
- 중간 길이
- 숫자 근거와 해석 균형

---

## 6. 연간 보고서 Prompt

### 6.1 목적

연간 회고 및 다음 해 전략 초안.

### 6.2 User Prompt Template

```text
아래 payload를 바탕으로 "연간 운영 전략 보고서"를 작성하라.

작성 목표:
- 올해 운영 체질을 한 문장으로 규정할 것
- 시즌별 변화, 가격 정책, 할인 정책, 공급 전략을 구분해서 해석할 것
- 내년 전략 제안은 실행 가능한 수준으로 정리할 것
- 경영진이 읽는 문서처럼 간결하고 구조적으로 작성할 것

출력 형식:
1. 연간 총평
2. 시즌별 해석
3. 가격/할인 정책 회고
4. 공급 전략 회고
5. 내년 전략 제안

payload:
{payload_json}
```

### 6.3 출력 톤

- 경영진 보고형
- 단정 대신 근거 기반 서술

---

## 7. Fallback 정책

LLM이 없거나 실패하면 deterministic renderer가 아래를 대신 수행한다.

- summary: 코드 기반 텍스트 조립
- actions: priority 순 정렬 후 직접 출력
- risks: 데이터 부족/누락 메시지 직접 출력
- recommendations: action 기반 문장 조립

즉, 프롬프트는 품질 향상용이며 필수 경로가 아니다.

---

## 8. 구현 메모

- `llm_report_writer.py`는 이 문서를 그대로 참조한다.
- 프롬프트는 코드 하드코딩보다 템플릿 상수/파일 분리 구조가 낫다.
- 일간 → 주간 → 월간 → 연간 순서로 적용한다.
